## Scope

This reference covers the workshop/runtime configuration surface used by:

- `packages/agentic_runtime/src/agentic_runtime/settings.py`
- `.env.example`
- `packages/core/src/core/settings.py`
- workshop-specific env lookups used by the current codebase

Environment variable names follow the uppercase field names used by `pydantic-settings`.

## Runtime Models And Generation

| Variable | Type | Default | Used by | Notes |
| --- | --- | --- | --- | --- |
| `MODEL_NAME` | `str` | `Qwen/Qwen3.5-4B` | runtime, Lab 6 summary | Main text model. |
| `ORCHESTRATION_MODEL_NAME` | `str` | `Qwen/Qwen3.5-2B` | Lab 6 planner | Used for planning/orchestration. |
| `THINKINK_MODEL` | `str` | `Qwen/Qwen3.5-9B` | runtime | Typo preserved from code. Supported as-is. |
| `VISUAL_MODEL` | `str` | `lmstudio-community/Qwen3-VL-4B-Thinking-MLX-8bit` | vision flows | Default VLM setting. |
| `IMAGE_MODEL_NAME` | `str` | `flux2-klein-9b` | image generation flow | Used when the image mode is enabled. |
| `IMAGE_MODEL_QUANTIZE` | `int` | `8` | image generation flow | Quantization level. |
| `IMAGE_WIDTH` | `int` | `1024` | image generation flow | Output width in pixels. |
| `IMAGE_HEIGHT` | `int` | `1024` | image generation flow | Output height in pixels. |
| `IMAGE_STEPS` | `int` | `4` | image generation flow | Sampling/inference steps. |
| `IMAGE_OUTPUT_DIR` | `str \| null` | `None` | image generation flow | Optional output directory override. |

## Storage, Streaming, And API Access

| Variable | Type | Default | Used by | Notes |
| --- | --- | --- | --- | --- |
| `MESSAGE_STORE_PATH` | `str \| null` | `None` | runtime | Optional message store path. |
| `MESSAGE_STORE_BATCH_SIZE` | `int` | `64` | runtime | Batch size for persisted message writes. |
| `MESSAGE_STORE_FLUSH_INTERVAL_SECONDS` | `float` | `0.05` | runtime | Flush interval for message persistence. |
| `MESSAGE_STREAM_INLINE_BYTES` | `int` | `4096` | runtime | Inline payload size threshold. |

| `API_TIMEOUT` | `float` | `120.0` | API-backed models | Request timeout in seconds. |
| `CHAT_MODEL_NAME` | `str` | `""` | distributed/local chat | If empty, chat mode reports that the chat model is not configured. |
| `CHAT_SERVER_NAME` | `str` | `127.0.0.1` | chat UI | Bind host for the chat server. |

| `REDIS_STREAM_PREFIX` | `str` | `agentic` | distributed runtime | Prefix for control, health, and message streams. |

## Minimal `.env` For Lab 6

```dotenv
API_BASE_URL=http://localhost:1234
DOCKER_API_BASE_URL=http://host.docker.internal:1234
API_KEY=
MODEL_NAME=Qwen/Qwen3.5-4B
ORCHESTRATION_MODEL_NAME=Qwen/Qwen3.5-2B
AGENTIC_TRANSPORT=redis_streams
REDIS_URL=redis://redis:6379/0
REDIS_STREAM_PREFIX=agentic
CHAT_ENTRY_AGENT=planner
DISTRIBUTED_CHAT_SOURCE=chat
DISTRIBUTED_CHAT_TIMEOUT_SECONDS=60
CHAT_SERVER_NAME=0.0.0.0
CHAT_SERVER_PORT=7860
AGENT_HEARTBEAT_SECONDS=5
AGENT_LIVENESS_TTL_SECONDS=20
LANGSEARCH_API_KEY=
LANGSEARCH_BASE_URL=https://api.langsearch.com
LANGSEARCH_TIMEOUT=20
LAB6_SEARCH_RESULTS_PER_QUERY=5
LAB6_SUMMARY_MAX_RESULTS=6
LAB6_SUMMARY_SNIPPET_CHARS=600
LAB6_SUMMARY_TOTAL_CHARS=6000
```

## Notes

- The current code uses `THINKINK_MODEL` exactly with that spelling. Do not silently rename it in
  configuration.
- `AGENT_SERVICE_NAME` is empty by default because it only makes sense for a dedicated service
  process.
- The workshop docs focus on runtime and lab configuration. Evaluation-only and dataset-only env
  toggles elsewhere in the repo are intentionally out of scope here.
